How to create hexo blog(7)?

前期准备工具

Git,Node.js,具体资料见文档最后的参考和资料

使用以下命令，出现版本信息则安装无误

node -v
npm -v

安装cnpn 使用国内镜像，后边操作更快

npm install -g cnpm --registry = "https://registry.npm.taobao.org/"

或者直接给npm换源，在npmrc文件中，直接粘贴如下内容

registry = "https://registry.npm.taobao.org/"

本人电脑上npmrc文件的路径C:\Users\lenovo\AppData\Roaming\npm\node_modules\npm，和node的安装路径有关

可能有用的操作

npmrc文件可修改内容：

registry = "https://registry.npm.taobao.org/"
prefix = C:\Program Files\nodejs
cache = C:\Program Files\nodejs\node_cache
说明：registry是下载源，为淘宝npm镜像仓库，prefix是全局下载的路径，cache是缓存位置了。
刚才的缓存文件夹要新建，即在nodejs下新建node_cache文件夹。

使用 cnpm 全局安装hexo

cnpm install -g hexo-cli

最好不要修改默认的安装位置，不然可能安装后无法使用

hexo -v 验证是否安装成功

创建本地博客

在自己想要放置博客的路径下，创建文件夹

mkdir MerlinDustBlog

该目录可在任意路径下

初始化

hexo init

启动

hexo server
localhost:4000

创建文章

hexo new "文章名"

重新生成

hexo clean  清除缓存文件 db.json 和已生成的静态文件 public 。
hexo generate

免费的部署方式 地址：用户名.github.io

创建一个叫 用户名.github.io的仓库

安装github部署插件

cnpm install --save hexo-deployer-git

在_config.yml 文件中设置(最后)

deploy:
    type: git
    repo: https://github.com/DustMerlin/DustMerlin.github.io.git
    branch: master

注意：符号为英文符号，且:之后有一个空格

部署到远端 

hexo deploy

通过url:用户名.github.io 访问 hexo 博客
